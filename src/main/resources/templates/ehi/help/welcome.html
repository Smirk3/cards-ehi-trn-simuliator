About application
WEB Application is used to simulate card authorization messages of GPS provider (https://globalprocessing.net/) like GPS External Host Interface (EHI) do.
External Host Interface (EHI) is a real-time transactional data feed that enables you to implement bespoke decision-making logic and your own authorization rules.
Getting started
1. Configuration
  • Go to settings page and click "Use default settings" button.
  • Check application variables
  • Go to list of cards page and add cards you want to do request for.
Do requests
  • Fill in main request parameters on messages page and click "Next" button.
  • You will see automatically generated XML request body. You can change this XML document manually by clicking "Edit" button on the top right corner of XML document frame and then click "Save" button.
  • Click "Submit" button on the bottom of XML document
  • On the next page yuo will see:
    • First block: response message and code. Green background - success, reg - failed
    • Second block: table of requests. Click on table raw and yuo will see XML request and response.
    • Third block: buttons to do next notification request based on recently done. There are implemented several external host notification flows. Application suggests yuo to do all available notifications based on recently done or do new notification message to start new flow of notification messages.
